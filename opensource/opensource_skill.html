AI Skill 设计方法论（2026 业界最佳实践）
🧘
Skill 之禅
复杂即功能（Skill 是文件夹非单文件）· 隐式激活（模式匹配而非显式调用）· 信号密度最大化（每个 Token 都必须有价值）· 重视踩坑点（Gotchas 是核心价值）
🪜
渐进式披露
description 定义触发边界 → SKILL.md 写核心流程 → references/ 放参考资料 → scripts/ 放执行脚本。减少无关信息进入上下文，降低 Token 消耗。
🎛️
自由度分级
高自由度（分析诊断）→ 启发式策略 · 中等自由度（报告生成）→ 模板框架 · 低自由度（格式转换）→ 固定脚本+明确顺序。根据容错率控制约束强度。
🚧
边界清晰
Skill ≠ Prompt（临时探索）· Skill ≠ 脚本（确定性执行）· Skill ≠ 知识库（存储内容）· Skill ≠ MCP（提供操作能力）。Skill = 任务编排器，告诉 Agent 怎么做。
三层上下文成本模型（Token 经济学）
Index 层（寸土寸金）
仅包含 name + description，每次对话都加载，必须极精简
~100 tokens
Load 层（精打细算）
SKILL.md 主体内容，加载时付费，聚焦核心流程和 Gotchas
~5,000 tokens
Runtime 层（按需自由）
scripts/、references/ 重资产，仅 Agent 实际读取时才消耗
按需加载

/goal 最佳实践：可验证目标驱动
核心模型
把 Agent 从“执行下一条指令”升级为“达成一个可验证完成契约”
/goal 适合长任务：目标持续挂在线程上，Codex 在每轮之后用测试、日志、benchmark、截图、报告或产物判断是否完成。它不是无限循环，而是“目标、证据、边界、停止条件”的组合。 
/goal 达成 <明确终态>，由 <测试/基准/日志/产物/报告> 验证，同时保持 <不可破坏约束>。
仅使用 <允许的文件/工具/数据/边界>。每轮迭代记录 <变更、证据、下一步>。
如果无法验证或没有合理路径，停止并报告 <尝试过什么、证据、阻塞点、需要的输入>。
适合使用
迁移、性能优化、flaky test、bug 复现、提示词/eval 优化、研究复现、多步骤原型。共同点是：目标明确、验证面明确、路径需要多轮探索。
不适合使用
一行修改、简单解释、短代码审查、互不相关的任务清单，以及“让它更好”“优化一下”这类没有完成标准的开放式请求。
六个要素
Outcome、Verification surface、Constraints、Boundaries、Iteration policy、Blocked stop condition。缺少验证面或阻塞停止条件的目标最容易跑偏。
监督方式
用 checkpoint 看当前状态、已验证证据、剩余工作和阻塞点。跑偏时修改目标契约，不要不断追加零散提示。
弱目标 强目标
/goal Improve performance 降低 checkout p95 到 120 ms 以下，由 checkout benchmark 验证，同时保持 correctness suite 绿色；每轮记录变更、指标和下一步。
/goal Fix flaky test 复现并修复指定 flaky test，目标测试本地连续 20 次通过，相关 suite 通过；不得跳过测试或削弱断言。
/goal Write docs 生成覆盖生命周期、命令面和两个示例的文档页；本地构建通过，所有命令与当前 CLI 行为一致。
/goal Reproduce this paper 建立 claim inventory，尝试可行 headline results，最终报告区分已确认机制、近似复现、阻塞声明和剩余不确定性。

推荐工作流
先 /plan 收敛目标，再 /goal 执行；长背景放 PLAN.md 或 program.md
命令速查
/goal、/goal pause、/goal resume、/goal clear
最佳示例
适合 PLAN.md 驱动的多步骤实现
/goal Implement PLAN.md into a working first version. Verify each milestone with the listed tests or Playwright checks. Keep changes scoped to the files named in PLAN.md unless a dependency is clearly required. After each checkpoint, record what changed, what passed, what failed, and the next smallest useful step. Stop only when all acceptance criteria pass, or when blocked with evidence and the exact input needed.
