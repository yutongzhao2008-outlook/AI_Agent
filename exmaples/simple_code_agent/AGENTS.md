# Repository Guidelines

## Project Structure & Module Organization

This is a minimal TypeScript Node.js code agent. Source lives in `src/`: `index.ts` starts the interactive CLI, `agent.ts` owns the Anthropic ReAct loop, and `tools.ts` defines and executes file/shell tools. Unit tests are under `src/__tests__/` and currently focus on tool behavior. `hello_world/` contains small sample files for agent experiments. `dist/` is generated by the TypeScript build and should not be edited directly.

## Build, Test, and Development Commands

- `npm install`: install runtime and development dependencies.
- `npm run dev`: run the CLI directly from `src/index.ts` with `ts-node`.
- `npm run build`: compile TypeScript into `dist/` using `tsc`.
- `npm start`: run the compiled app from `dist/index.js`; run `npm run build` first.
- `npm test`: run Jest tests using `ts-jest`.

## Coding Style & Naming Conventions

Use strict TypeScript with CommonJS modules, matching `tsconfig.json`. Follow the existing style: two-space indentation, semicolons, single quotes, explicit exports, and clear type annotations where values cross module boundaries. Name source files by purpose, for example `agent.ts`, `tools.ts`, and `*.test.ts`. Keep tool names stable and snake_case when exposed to the LLM, such as `read_file` or `run_bash`.

## Testing Guidelines

Jest is configured to discover tests matching `src/__tests__/**/*.test.ts`. Add or update tests when changing tool behavior, agent control flow, or user-visible CLI behavior. Prefer focused unit tests that avoid real API calls; the existing tests use temporary directories for filesystem side effects. Run `npm test` before submitting changes, and run `npm run build` when TypeScript types or exports change.

## Commit & Pull Request Guidelines

Recent history uses short, imperative commit summaries such as `add ...` and `init commit`. Keep commit messages concise and describe the main change in the first line. Pull requests should include a brief purpose statement, notable implementation details, test/build results, and any linked issue or follow-up. Include terminal output or screenshots only when they clarify behavior.

## Security & Configuration Tips

Copy `.env.example` to `.env` and set `ANTHROPIC_API_KEY` locally. Do not commit `.env`, API keys, debug logs, or generated secrets. Be careful when changing `run_bash` or file-writing behavior; these tools execute local operations on behalf of the agent.
